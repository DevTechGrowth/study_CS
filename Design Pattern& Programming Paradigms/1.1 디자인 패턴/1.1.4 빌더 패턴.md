**빌더 패턴**은 복잡한 객체들을 단계별로 생성할 수 있도록 하는 생성 디자인 패턴

1. 필요한 데이터만 설정할 수 있다.
2. 유연성을 확보할 수 있다.
3. 가독성을 높일 수 있다.

``` java
public class User { 
    private String name;
    private int age;
    private int height;
    private int iq;

    public User (String name, iny age, int height, int iq) {
        this.name = name;
        this.age = age;
        this.height = height;
        this.iq = iq;
    }
    
    public UserBuilder builder() {
        return new UserBuilder();    
    }
}

public class UserBuilder {
    private String name;
    private int age;
    private int height;
    private int iq;
    
    public UserBuilder id(String name) {
        this.name = name;
        return this;
    }

    public UserBuilder age(int age) {
        this.age = age;
        return this;
    }

    public UserBuilder height(int height) {
        this.height = height;
        return this;
    }

    public UserBuilder iq(int iq) {
        this.iq = iq;
        return this;
    }
    
    public User build() {
        return new User(name, age, height, iq);
    }
}
```
